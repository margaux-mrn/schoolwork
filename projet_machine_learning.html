In the Machine Learning project, we chose to work on image recognition of the sign language alphabet.

The database used comes from Kaggle, and provides us with two .csv files (one for training and one for testing) that contain the pixel description of each image.

The training file provides data for 27455 images and the test file for 7172 images. The letters J and Z are not represented in the image because they are movements.

Each image consists of 784 pixels with a value from 0 to 255, corresponding to the gray level of the pixel.

The images correspond to the rows of the data tables, they are called by a label ranging from 0 (for the letter A) to 24 (for the letter Y), and the pixels of the images correspond to the columns.

def reshape_to_2d(data, dim):
    reshaped = []
    labs = data[:,0]
    data = data[:,1:]
    for row in data :
        reshaped.append( row.reshape(dim, dim) )
    return ( labs,np.array(reshaped) )

Kmeans¶

Use of scikit-learn package for the use of the Kmeans algorithm

The k-means method is a data partitioning method.

The k-means are used in unsupervised learning where observations are divided into k partitions. Given points and an integer k, the problem is to divide the points into k groups, often called clusters, in order to minimize a certain function. We consider the distance from a point to the average of the points in its cluster; the function to be minimized is the sum of the squares of these distances.

DSCAN¶

The DBSCAN (density-based spatial clustering of applications with noise) algorithm uses two parameters: distance $\epsilon$ and the minimum number of points MinPts that must be within a radius $\epsilon$ for those points to be considered a cluster.

The input parameters are therefore an estimate of the point density of the clusters.The basic idea of the algorithm is then, for a given point, to retrieve its $\epsilon$-neighborhood and check that it contains MinPts points or more.

That point is then considered part of a cluster. The $\epsilon$-neighborhood is then scanned step by step in order to find all the points of the cluster.

Spectral Clustering¶

Spectral cluste is a K-group partitioning method based on minimizing a "slice" criterion. These two measures express the internal cohesion of the groups, relative to their dissociation from each other. They are directly functions of a matrix of similarities between objects, noted S.

Given a set of N objects, it is possible to obtain a detailed representation of this set in the form of a weighted graph, noted G(V,E,S). V denotes the set of N nodes of the graph, corresponding to the objects; E is the set of inter-node arcs and S is the weight matrix of the arcs (similarity matrix), symmetrical and non-negative.

We use the test set for this clustering, because the training set doesn't work, as it is probably too heavy.

Part 2 : SVM¶

In this part of the project we are going to use de SVM methods from the sk_learn library. Several methods are available with different kernels end different separation scheme.

First we are looking at the two ways we can separate data: "one vs one" and "one vs the rest". On the one hand we are creating (N_class*(N_class-1))/2 classifiers looking at every couple of categories. We then aggregate the results to create the final classifier. On the second hand we are using the "one vs the rest" method, creating N_class classifiers considering each category in front of the rest of the data and the, as before, aggregating the classifiers.

These two methods lead to the same results in the end. In fact the structures that are caught with the two methods are the same resulting in the same score.

We then look at several kernel definitions, firstly a sigmoid kernel and then a polynomial kernel.

Decision Tree¶

We train decision trees of different depths over the train sample, using the Gini criterion at each node. We then evaluate the optimal depth by looking at the error over the validation sample.

The construction of a binary discrimination tree consists in the determination of a sequence of nodes. A node is defined by the joint choice of a variable among the explanatory and a division which induces a partition into two classes. Implicitly, each node therefore corresponds to a subset of the sample to which a dichotomy is applied. A division is defined by a threshold value of the quantitative variable selected or a division in two groups of modalities if the variable is qualitative. The root or initial node corresponds to the entire sample; the the procedure is then iterated over each of the subsets.

Boosting¶

We generate a boosting model using decision trees. The idea is to create several subsets of data from a randomly selected training sample with replacement. Each subset data collection is then used for old decision trees. As a result, we end up with a set of different models. The average of all the predictions from different trees is used, which is more robust than a single decision tree.

def many_parameters_four_cnn(num_classes):
  # code à réaliser dans le tp sur les reseaux de neurones pour l'exemple des pickles
  model = Sequential()
  model.add(Conv2D(32, kernel_size=(5,5), activation = 'relu', input_shape=(28, 28 ,1) ))
  model.add(MaxPooling2D(pool_size = (2, 2)))

  model.add(Conv2D(64, kernel_size = (5, 5), activation = 'relu'))
  model.add(MaxPooling2D(pool_size = (2, 2)))
  model.add(Dropout(0.25))


  model.add(Flatten())
  model.add(Dense(1024))
  model.add(Dropout(0.5))

  model.add(Dense(num_classes, activation = 'softmax'))

  model.compile(
    loss = losses.categorical_crossentropy,
    optimizer = optimizers.Adagrad(),
    metrics = ['accuracy'] )
  
  return model
